01 — Fundamentals
What is a Stack?
📚

LIFO Principle

Last In, First Out. Like a stack of plates — you always add and remove from the top. The last element pushed is the first one popped.

⚡

Core Operations

push(x) — add to top  |  pop() — remove top  |  peek() — view top  |  isEmpty() — check empty. All O(1).

🔍

When to Use

Whenever you need to remember previous state to handle future elements — matching brackets, tracking history, monotonic sequences, or DFS traversal.

🧠

Mental Model

Think of a stack as an undo buffer. You store things you haven't fully "resolved" yet, and pop them when you find their match or answer.

04 — Tips & Tricks
Master the Stack Technique
TIP 01

The "Unresolved" Pattern

Push elements onto the stack when you can't yet determine their answer. Pop them off when you finally find the information you need.

// waiting for closing bracket stack.push(char) // found it → resolve if match(stack.top, char): stack.pop()
TIP 02

Monotonic Stack

Maintain a stack in monotonically increasing or decreasing order. When the order breaks, pop and record answers. Used for "next/previous greater/smaller" problems.

for num in arr: while stack and stack[-1] < num: result[stack.pop()] = num stack.append(i)
TIP 03

Push Index, Not Value

Often you need both the value AND position. Push the index onto the stack — you can always retrieve the value as arr[stack[-1]].

stack = [] # push indices for i, val in enumerate(arr): while stack and arr[stack[-1]] < val: idx = stack.pop() ans[idx] = val stack.append(i)
TIP 04

Stack for DFS / Tree Traversal

Iterative DFS uses an explicit stack instead of recursion. This avoids stack overflow on deep inputs and gives you fine-grained control over traversal order.

stack = [root] while stack: node = stack.pop() # process node if node.right: stack.append(node.right) if node.left: stack.append(node.left)
TIP 05

Two-Stack Trick

Use two stacks to implement a queue (push stack + pop stack). Or use a "min stack" — a second stack that tracks the running minimum at each level.

class MinStack: def push(self, val): self.stack.append(val) mn = min(val, self.mins[-1]) self.mins.append(mn)
TIP 06

Duplicate Element on Stack

Sometimes push the same element twice (e.g., for inorder traversal simulation) to distinguish between "first visit" and "process now" states. Use a flag or sentinel.

stack = [(root, False)] while stack: node, visited = stack.pop() if visited: process(node) else: stack.append((node, True)) ...

06 — Complexity
Time & Space Analysis

Time Complexity

push(x)O(1)
pop()O(1)
peek()O(1)
isEmpty()O(1)
Monotonic traversalO(n)*

* Each element is pushed and popped at most once → O(n) amortized total, not O(n²).

Space Complexity

Stack storageO(n)
Best case (sorted)O(1)
Worst caseO(n)
Two-stack queueO(n)
Min stack (auxiliary)O(n)

Space is at most O(n) — you never store more elements than exist in the input.
