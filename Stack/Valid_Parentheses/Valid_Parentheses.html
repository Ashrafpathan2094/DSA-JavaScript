Fundamentals

1 What Is a Stack?

A stack is a linear data structure operating on the LIFO principle — Last In, First Out. The most recently added element is always the first to be removed.

Real-world analogies: a stack of plates, browser back-button history, undo in a text editor, the call stack in your runtime.

← PUSH / POP only here (TOP)
  '('   ← index 2 · TOP
  '['   index 1
  '{'   index 0 · bottom
🔑 Core Rule
You can only touch the top element. No random access. This single constraint makes the stack elegant for matching, backtracking, and undo problems.

3 JavaScript Implementation

// Using an Array (interview standard)

JSconst stack = []; stack.push('a'); // push → ['a'] stack.push('b'); // push → ['a','b'] stack[stack.length - 1]; // peek → 'b' stack.pop(); // pop → returns 'b', stack=['a'] stack.length === 0; // isEmpty → false

// Class-based Implementation

JSclass Stack { constructor() { this.data = []; } push(x) { this.data.push(x); } pop() { return this.data.pop(); } peek() { return this.data.at(-1); } isEmpty() { return this.data.length === 0; } size() { return this.data.length; } }

4 When to Reach for a Stack

Problem SignalWhy Stack Works
Matching pairs / bracketsPush openers, pop on closer, verify match
"Most recent" or "previous" elementStack gives O(1) access to last-seen item
Undo / backtrackLIFO naturally reverses operations
Nested structures (HTML, JSON, parens)Depth tracking via push/pop
Next Greater / Smaller ElementMonotonic stack pattern

Iterative DFS on graph/treeExplicit stack replaces recursive call stack
⚠ Watch Out
Always check isEmpty() before calling pop() or peek(). In JavaScript, popping an empty array returns undefined — not an error — which can silently corrupt comparisons.
💡 JS Trick
stack.pop() returns the removed element. Use this to check-and-remove in one line:
if (stack.pop() !== expected) return false;

5 Complexity Summary

push: O(1) pop: O(1) peek: O(1) space: O(n) worst case

Worst-case space: every character is an opener (e.g. "((((("), so all n chars are on the stack before any pops happen.

Stack Patterns

1 Matching Brackets

The most classic stack pattern. Push every opener. On every closer, pop and verify the match. At end, the stack must be empty.

Template Logic
Push openers → closer seen? pop and compare → end of string? check stack is empty.
JS · Matching Templateconst map = { ')':'(', ']':'[', '}':'{' }; const stack = []; for (const ch of s) { if (map[ch]) { if (stack.pop() !== map[ch]) return false; } else { stack.push(ch); } } return stack.length === 0;

2 Monotonic Stack

Maintain a stack in strictly increasing or decreasing order. When a new element violates the order, pop until it doesn't. Perfect for "next greater/smaller" problems.

JS · Next Greater Elementfunction nextGreater(nums) { const res = new Array(nums.length).fill(-1); const stack = []; // stores indices for (let i = 0; i < nums.length; i++) { while (stack.length && nums[stack.at(-1)] < nums[i]) { res[stack.pop()] = nums[i]; } stack.push(i); } return res; }

3 Stack for Iterative DFS

Replace the implicit call stack with an explicit one. Push neighbours, pop to visit. Identical logic to recursion, but avoids stack overflow on huge graphs.

JS · Iterative DFSfunction dfs(graph, start) { const visited = new Set(); const stack = [start]; while (stack.length) { const node = stack.pop(); if (visited.has(node)) continue; visited.add(node); for (const nb of graph[node]) stack.push(nb); } }

Code Analysis

1 Your Original Code

JS · Originalvar isValid = function(s) { let stack = []; const closingObj = { ")":"(", "}":"{", "]":"[" }; for (let i = 0; i < s.length; i++) { if (s[i] === "(" || s[i] === "[" || s[i] === "{") { stack.push(s[i]); } else { const topElement = stack[stack.length - 1]; if (topElement != closingObj[s[i]]) { return false; } else { stack.pop(); } } } if (stack.length) return false; return true; };

2 Optimised Code

Change 1 — Detect closers via map lookup (truthy check)
Replaced the verbose s[i]==='(' || s[i]==='[' || ... chain with map[ch]. If truthy it's a closer, if falsy (undefined) it's an opener. Also safely handles unexpected characters.
Change 2 — Combine peek + pop into one call
stack[stack.length-1] then separately stack.pop() = two operations. Replaced with a single stack.pop() directly in the condition. Cleaner, fewer lines, same result.
Change 3 — Single-expression final return
if (stack.length) return false; return true; collapses to return stack.length === 0;
JS · Optimisedvar isValid = function(s) { const stack = []; const map = { ')':'(', '}':'{', ']':'[' }; for (const ch of s) { if (map[ch]) { if (stack.pop() !== map[ch]) return false; } else { stack.push(ch); } } return stack.length === 0; };
Time: O(n) Space: O(n) Same complexity · Cleaner code
