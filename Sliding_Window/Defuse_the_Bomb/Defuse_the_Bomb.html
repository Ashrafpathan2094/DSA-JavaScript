01 Problem

Given a circular array code and integer k, replace each element with the sum of the next or previous k elements (treating the array as circular).

k > 0

Replace code[i] with sum of next k elements → window is [i+1 … i+k]

k < 0

Replace code[i] with sum of prev |k| elements → window is [i+k … i-1]

k = 0

Every element becomes 0. Trivial base case.

02 Naive vs Optimal
✗ BRUTE FORCE — O(n·k)

For each index i, loop k times summing elements. The window is recomputed from scratch every single time — wasteful.

✓ SLIDING WINDOW — O(n)

Compute window once. Each step: subtract one leaving element, add one entering element. Window slides in O(1) per step.

03 Algorithm Walkthrough
1

Handle k = 0 immediately → return all zeros.

2

Determine initial window bounds for index 0:
k > 0 → start = 1, end = k
k < 0 → start = n+k, end = n-1

3

Sum the initial window using modulo for circular wrapping: code[i % n]

4

Record res[0] = windowSum

5

For each i from 1 to n-1:
→ Subtract code[start % n] (element leaving window)
→ Add code[(end+1) % n] (element entering window)
→ Advance start and end by 1
→ Record res[i] = windowSum

04 Key Insight — Unified Window

The elegant insight is that negative k is just a left window — expressible as a right-shifted start index. Instead of writing two separate loops, map both cases to the same start/end pointer logic:

K POSITIVE
start = 1
end = k
K NEGATIVE
start = n + k
end = n - 1

After that, the sliding loop is identical for both cases. This eliminates the duplicated code your original solution had.

OPTIMAL Solution
// Unified sliding window — O(n) time · O(n) space var decrypt = function(code, k) { const n = code.length; const res = new Array(n).fill(0); // Base case — k=0, all zeros if (k === 0) return res; // Determine initial window for index 0: // k > 0 → window = [1 … k] (next k elements) // k < 0 → window = [n+k … n-1] (prev |k| elements) let [start, end] = k > 0 ? [1, k] : [n + k, n - 1]; // Build initial window sum (circular via modulo) let windowSum = 0; for (let i = start; i <= end; i++) { windowSum += code[i % n]; } // Slide the window across all n positions for (let i = 0; i < n; i++) { res[i] = windowSum; windowSum -= code[start % n]; // ← remove leaving element windowSum += code[(end + 1) % n]; // ← add entering element start++; end++; } return res; };
WHY This Beats Your Original
→

Your solution had two completely separate loops for k>0 and k<0. The optimal has one unified loop.

→

The trick: negative k's left-window maps to a right-shifted start index (n + k), so the sliding logic is identical.

→

Both have the same O(n) complexity — the improvement is purely in code clarity and maintainability.
