Max Consecutive Ones III
Longest Ones

Binary array · flip at most k zeros · find longest window of 1s
📋 Problem Statement
Given a binary array nums and an integer k, return the maximum number of consecutive 1s in the array if you can flip at most k zeros.
Sliding Window Two Pointers Medium
💡 Core Intuition
🪟

Sliding Window

Maintain a window [left…right] that contains at most k zeros. Grow right each step.

↔️

Two Pointers

When zeros in window > k, shrink from left until the count is valid again.

📏

Track Max

At every step, window size = right − left + 1. Update max continuously.

🔄

Never Shrink Smaller

We only shift left by 1 when zeros > k, keeping the window size non-decreasing. This is the key insight!

🧠 Algorithm — Step by Step
  1. Initialise left = 0, zeros = 0, max = 0.
  2. Iterate right from 0 to n−1 (expand the window one step at a time).
  3. If nums[right] === 0, increment zeros.
  4. If zeros > k — window is invalid! Check nums[left]: if it's a 0, decrement zeros. Either way, advance left++.
  5. Update max = Math.max(max, right − left + 1).
  6. Return max.

💻 Annotated Code
var longestOnes = function (nums, k) {
  let left  = 0;   // left pointer of window
  let max   = 0;   // best window size seen
  let zeros = 0;   // zeros inside current window

  for (let right = 0; right < nums.length; right++) {
    // expand window: include nums[right]
    if (nums[right] === 0) zeros++;

    // window invalid → shrink from left by exactly 1
    if (zeros > k) {
      if (nums[left] === 0) zeros--; // evict a zero
      left++;
    }

    // window always grows or stays same size → safe to max every step
    max = Math.max(max, right - left + 1);
  }
  return max;
};
📊 Complexity
O(n)
Time — each element visited at most twice (once by right, once by left)
O(1)
Space — only 3 variables, no extra data structures
⚠️ Common Gotchas
🔄
Don't reset left to 0 — we only ever increment left, never jump it backward. The window only slides forward.
1️⃣
Move left by exactly 1 — even if zeros is still > k after moving, we don't loop. The next right iteration will handle it.
🪟
Window never shrinks below its max — this is the elegant insight that lets us update max at every step with right − left + 1.
🎯
k=0 is valid — the algorithm handles it naturally; no zeros are allowed in the window.
